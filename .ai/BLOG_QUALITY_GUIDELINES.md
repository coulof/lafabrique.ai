# Blog Post Quality Guidelines

## General Structure

### Front Matter
- **Date**: Should be in YYYY-MM-DD format
- **Authors**: Should reference valid authors from `.authors.yml`
- **Categories**: Should be relevant and consistent (e.g., Kubernetes, DevOps, AI, Storage)
- **Title**: Should be clear, concise, and engaging

### Content Structure
1. **Introduction**: Clear problem statement or context
2. **Main Content**: Well-organized with logical flow
3. **Conclusion**: Summary and key takeaways
4. **Call to Action**: Links to related content or resources

## Writing Style

### Language & Grammar
- Use American English spelling and grammar
- Write in active voice where possible
- Keep sentences concise (aim for 15-20 words max)
- Use bullet points and lists for complex information
- Avoid jargon unless necessary, and explain technical terms

### Technical Content
- Code snippets should be properly formatted with syntax highlighting
- Commands should be clearly marked and testable
- Include version information for tools/frameworks mentioned
- Provide context for why certain approaches are used
- Link to official documentation where relevant

### Formatting
- Use proper Markdown headers (##, ###, ####)
- Use `<!-- more -->` tag for excerpt separation
- Images should have descriptive alt text and captions
- Use emojis and icons sparingly and consistently ; use mkdocs-material-icons when possible
- External links should open in new tabs
- Use regular quotes such as ' "
- Do not use Em dash

## SEO & Readability

### Metadata
- Title should be 60 characters or less
- Include relevant keywords naturally
- Use descriptive, keyword-rich headings

### Readability
- Aim for Flesch Reading Ease score of 60-70
- Use subheadings every 300 words or so
- Break up long paragraphs (3-4 sentences max)
- Use visual elements (images, diagrams, code blocks)

## Technical Requirements

### Code Examples
- All code snippets should be tested
- Include error handling where relevant
- Show both the command and expected output
- Use consistent indentation (2 or 4 spaces)

### Links
- All external links should use HTTPS
- Internal links should use relative paths
- Check for broken links regularly
- Use descriptive link text (avoid "click here")

## Review Checklist

- [ ] Front matter is complete and correct
- [ ] Title is engaging and descriptive
- [ ] Introduction clearly states the purpose
- [ ] Content flows logically
- [ ] Technical information is accurate
- [ ] Code examples work as shown
- [ ] Images have proper alt text
- [ ] Links are working and relevant
- [ ] Conclusion summarizes key points
- [ ] Grammar and spelling are correct
- [ ] Formatting is consistent
- [ ] SEO elements are present
